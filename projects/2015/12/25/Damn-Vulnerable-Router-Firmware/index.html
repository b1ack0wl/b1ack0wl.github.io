Damn Vulnerable Router Firmware

by b1ack0wl — on Projects 25 Dec 2015

As microprocessors become cheaper and cheaper it appears that everything that doesn’t need a processor is getting a processor. Unfortunately a lot of these devices are programmed with poor programming practices which is then sold off to the consumer. To help prevent this trend I decided to start the “Damn Vulnerable Router Firmware” project. This project is based of DD-WRT technology stock firmware images but has been re-written in a way to make it vulnerable. This project has been inspired by the Damn Vulnerable Web App project since I believe the hands on experience is a greater teaching tool. Here are the following highlights of the project:

  • Support MIPS32be/le and multiple ARM Architectures.
  • Prevent users from accidentally installing this leaving their device wide open and vulnerable
  • Disable WAN DHCP (User will have to manually turn this on via CLI)
  • Display a huge warning sign when the user first enters the admin web interface
  • Vulnerabilities ranging from Web to Memory Corruption
  • Memory corruption will range from simple no mitigation exploitation to full memory protections (e.g ASLR, DEP, Stack Cookies..etc)
  • Support for encrypted or obfuscated firmware binaries to force hardware hacking (e.g Dumping firmware from flash memory)
  • Plus in no time you can be like this!

  • Potentially integrate DD-WRT as a main platform.


**Version v0.1b to be released on or around January 2016**
